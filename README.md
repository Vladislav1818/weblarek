# Проектная работа «Веб-ларёк»

Стек: HTML, SCSS, TypeScript, Vite

## Описание проекта

«Веб-ларёк» — это интернет-магазин с товарами для веб-разработчиков.  
Пользователь может:

- просматривать каталог товаров;
- открывать карточку товара в модальном окне;
- добавлять товары в корзину;
- удалять товары из корзины;
- заполнять данные для оформления заказа;
- отправлять заказ на сервер.

Проект реализован по архитектуре **MVP (Model-View-Presenter)**.

## Архитектура приложения

Код приложения разделён на слои согласно парадигме **MVP (Model-View-Presenter)**, которая обеспечивает разделение ответственности между частями приложения.

### Model
Слой данных. Отвечает за хранение, изменение, получение и валидацию данных.

### View
Слой представления. Отвечает за отображение данных в интерфейсе.

### Presenter
Связывает Model и View. Обрабатывает действия пользователя, изменяет данные в моделях и передаёт обновлённые данные во View.

Взаимодействие между слоями построено на событиях. Для этого используется брокер событий `EventEmitter`.

## Базовый код

### Класс `Component`

Является базовым классом для всех компонентов интерфейса.

Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

#### Конструктор

```ts
constructor(container: HTMLElement)
```

Принимает ссылку на DOM-элемент, за отображение которого отвечает компонент.

#### Поля

- `container: HTMLElement` — корневой DOM-элемент компонента.

#### Методы

- `render(data?: Partial<T>): HTMLElement` — главный метод класса. Принимает данные для отображения, записывает их в поля экземпляра и возвращает корневой DOM-элемент компонента.
- `setImage(element: HTMLImageElement, src: string, alt?: string): void` — утилитарный метод для установки изображения и альтернативного текста.

### Класс `Api`

Содержит базовую логику отправки HTTP-запросов.

#### Конструктор

```ts
constructor(baseUrl: string, options: RequestInit = {})
```

В конструктор передаётся базовый адрес сервера и объект параметров запроса.

#### Поля

- `baseUrl: string` — базовый адрес сервера.
- `options: RequestInit` — объект с параметрами запроса.

#### Методы

- `get(uri: string): Promise<object>` — выполняет GET-запрос на переданный эндпоинт и возвращает промис с объектом ответа.
- `post(uri: string, data: object, method?: ApiPostMethods): Promise<object>` — отправляет данные на сервер. По умолчанию использует метод `POST`, но может принимать и другие методы.
- `handleResponse(response: Response): Promise<object>` — защищённый метод, который проверяет ответ сервера и возвращает данные или отклонённый промис при ошибке.

### Класс `EventEmitter`

Реализует паттерн **«Наблюдатель»** и используется для связи слоя данных и слоя представления.

#### Конструктор

Класс не принимает параметров.

#### Поля

- `_events: Map<string | RegExp, Set<Function>>` — коллекция подписок на события.

#### Методы

- `on<T extends object>(event: EventName, callback: (data: T) => void): void` — подписка на событие.
- `emit<T extends object>(event: string, data?: T): void` — инициализация события.
- `trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` — возвращает функцию, которая вызывает нужное событие.

## Данные

В приложении используются следующие сущности данных.

### Тип оплаты

```ts
type TPayment = 'card' | 'cash';
```

Определяет способ оплаты заказа.

### Интерфейс товара

```ts
interface IProduct {
	id: string;
	description: string;
	image: string;
	title: string;
	category: string;
	price: number | null;
}
```

Описывает товар каталога интернет-магазина.

#### Поля

- `id` — уникальный идентификатор товара;
- `description` — описание товара;
- `image` — путь к изображению товара;
- `title` — название товара;
- `category` — категория товара;
- `price` — цена товара. Если значение `null`, товар недоступен для покупки.

### Интерфейс покупателя

```ts
interface IBuyer {
	payment: TPayment;
	email: string;
	phone: string;
	address: string;
}
```

Описывает данные покупателя, которые вводятся при оформлении заказа.

#### Поля

- `payment` — способ оплаты;
- `email` — электронная почта покупателя;
- `phone` — телефон покупателя;
- `address` — адрес доставки.

### Тип ошибок валидации

```ts
type TBuyerErrors = Partial<Record<keyof IBuyer, string>>;
```

Тип объекта с ошибками валидации полей покупателя.  
Ключом объекта является название поля, значением — текст ошибки.

Пример:

```ts
{
  payment: 'Не выбран способ оплаты',
  email: 'Укажите email'
}
```

### Ответ сервера со списком товаров

```ts
interface IProductsResponse {
	total: number;
	items: IProduct[];
}
```

Описывает объект, который приходит с сервера при запросе каталога товаров.

#### Поля

- `total` — общее количество товаров;
- `items` — массив товаров.

### Интерфейс заказа

```ts
interface IOrder extends IBuyer {
	items: string[];
	total: number;
}
```

Описывает объект заказа, который отправляется на сервер.

#### Поля

- `items` — массив идентификаторов товаров;
- `total` — общая стоимость заказа;
- остальные поля наследуются из `IBuyer`.

### Ответ сервера на заказ

```ts
interface IOrderResponse {
	id: string;
	total: number;
}
```

Описывает ответ сервера после успешного оформления заказа.

#### Поля

- `id` — идентификатор заказа;
- `total` — итоговая сумма заказа.

## Модели данных

### Класс `ProductsModel`

Класс отвечает за хранение каталога товаров и товара, выбранного для подробного просмотра.

#### Поля

- `items: IProduct[]` — массив всех товаров каталога;
- `preview: IProduct | null` — товар, выбранный для подробного просмотра.

#### Конструктор

Класс не принимает параметров.

#### Методы

- `setItems(items: IProduct[]): void` — сохраняет массив товаров.
- `getItems(): IProduct[]` — возвращает массив товаров.
- `getItemById(id: string): IProduct | undefined` — возвращает товар по его идентификатору.
- `setPreview(item: IProduct): void` — сохраняет товар для подробного просмотра.
- `getPreview(): IProduct | null` — возвращает товар для подробного просмотра.

### Класс `BasketModel`

Класс отвечает за хранение товаров, добавленных в корзину.

#### Поля

- `items: IProduct[]` — массив товаров в корзине.

#### Конструктор

Класс не принимает параметров.

#### Методы

- `getItems(): IProduct[]` — возвращает массив товаров в корзине.
- `addItem(item: IProduct): void` — добавляет товар в корзину, если его там ещё нет.
- `removeItem(item: IProduct): void` — удаляет товар из корзины.
- `clear(): void` — очищает корзину.
- `getTotal(): number` — возвращает общую стоимость товаров в корзине.
- `getCount(): number` — возвращает количество товаров в корзине.
- `hasItem(id: string): boolean` — проверяет наличие товара в корзине по его идентификатору.

### Класс `BuyerModel`

Класс отвечает за хранение данных покупателя и их валидацию.

#### Поля

- `data: Partial<IBuyer>` — объект с данными покупателя.

#### Конструктор

Класс не принимает параметров.

#### Методы

- `setData(data: Partial<IBuyer>): void` — сохраняет переданные данные покупателя, не удаляя уже сохранённые поля.
- `getData(): Partial<IBuyer>` — возвращает объект с данными покупателя.
- `clear(): void` — очищает данные покупателя.
- `validate(): TBuyerErrors` — валидирует все поля покупателя.
- `validateStepOne(): TBuyerErrors` — валидирует первый шаг оформления заказа (`payment`, `address`).
- `validateStepTwo(): TBuyerErrors` — валидирует второй шаг оформления заказа (`email`, `phone`).

Поле считается валидным, если оно не пустое.

## Слой коммуникации

### Класс `LarekApi`

Класс отвечает за взаимодействие приложения с сервером.

Использует композицию: принимает в конструктор объект, реализующий интерфейс `IApi`, и вызывает его методы `get` и `post`.

#### Поля

- `api: IApi` — объект для выполнения HTTP-запросов;
- `cdn: string` — базовый путь к изображениям товаров.

#### Конструктор

```ts
constructor(api: IApi, cdn: string)
```

#### Параметры конструктора

- `api: IApi` — экземпляр класса API;
- `cdn: string` — адрес CDN для изображений товаров.

#### Методы

- `getProducts(): Promise<IProductsResponse>` — выполняет GET-запрос на `/product/`, получает каталог товаров и формирует полные ссылки на изображения.
- `createOrder(order: IOrder): Promise<IOrderResponse>` — выполняет POST-запрос на `/order/`, отправляет данные заказа и возвращает ответ сервера.
